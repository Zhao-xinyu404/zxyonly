# 产品需求文档（PRD）— 微聊（WeiChat Lite）

## 一、项目概述

**产品名称**：微聊（WeiChat Lite）
**产品定位**：一款仿微信简化版的纯前端聊天网页应用，支持账号注册、登录、好友添加、一对一聊天等核心功能，底部四 Tab 导航栏，移动端优先设计。

## 二、目标用户

- 希望体验轻量化即时通讯界面的用户
- 学习/演示用的小型聊天应用场景

## 三、核心功能

### 3.1 账号系统
- **注册**：用户名、密码、昵称、头像（可选预设头像）
- **登录**：用户名 + 密码校验
- **登出**：清除当前会话
- 数据存储：localStorage（前端持久化，无后端）

### 3.2 四 Tab 主界面（底部导航）
1. **微信（聊天列表）**：展示已有会话，未读红点，进入聊天
2. **通讯录**：好友列表，添加好友入口，好友资料查看
3. **发现**：朋友圈样式占位 + 二维码名片
4. **我**：个人资料展示、编辑、登出

### 3.3 聊天功能
- 一对一文字聊天
- 消息气泡（自己右侧绿色，对方左侧白色）
- 消息时间戳
- 多账号间共享数据，可互相收发消息（同一浏览器内）

### 3.4 好友系统
- 通过用户名搜索添加好友
- 好友请求确认（简化为直接添加）
- 好友列表展示

## 四、非功能需求

- **响应式**：移动端优先，桌面端居中展示手机框
- **性能**：纯静态单文件，秒开
- **视觉**：微信经典绿（#07c160）+ 简洁现代风
- **兼容**：现代浏览器（Chrome/Edge/Safari/Firefox）

## 五、页面结构

```
┌─────────────────────────┐
│   顶部标题栏             │
├─────────────────────────┤
│                         │
│   内容区域（随 Tab 切换）│
│                         │
├─────────────────────────┤
│ 微信 │ 通讯录 │ 发现 │ 我 │  ← 底部四 Tab
└─────────────────────────┘
```

## 六、数据模型（localStorage）

- `weichat_users`：所有注册用户数组
- `weichat_session`：当前登录用户名
- `weichat_friends_<username>`：该用户好友列表
- `weichat_msgs_<userA>_<userB>`：两人之间的消息记录（按字典序命名 key）
